Тебе нужно фиксить мобильные стили, при этом не ломать десктопные.
Если на странице не импортирован Section, добавь его в начало списка импортов (из того же файла).
В Section нужно перенести h2 в атрибут title, а первый параграф (если есть) в атрибут description. Но если внутри параграфа только текст без каких-либо тегов и он не слишком длинный, то передай как атрибут description (строка).
Убери лишние классы и aria-labelledby у секций, каждая секция имеет свой id и ориентироваться надо на него.
В квизах по умолчанию не должно быть эмодзи, если не указано вручную. Стандартный эмодзи не должен быть там, если не указан явно - не показывать ничего. В ответах на квизы не долнжо быть эмодзи в качестве иконки.
В квизах, если в тексте есть заголовок, его нужно вынести в атрибут title.
Нормализуй размеры шрифтов, паддинги, марджины и границы по примеру уроков 2-4. Обычно используется шрифт 1.25rem.
Внутри Summary у контента не должно быть лишних границ и прочего.
В Summary текст контента должен иметь тот же размер, что и в Summary на других страницах, для консистентности.
Все стили в конечном итоге должны быть сгруппированы на всю глубину в соответствии с разметкой, по сути повторяя её структуру. Длинные селекторы недопустимы: вместо .foo .bar .baz должно быть .foo { .bar { .baz { } } }. Всегда группируй на всю глубину вложенности, даже если там один потомок.
Для секции с кризисом id всегда "crisis", для summary - "summary".
Если какая-то группа элементов должна быть в одну строку, то будь уверен, что на мобилках оно сломается и будет либо в столбик, либо как попало. Превентивно почини мобильные стили в таких местах, не дожидаясь дополнительных комментариев.
Не ломай десктопные стили, после выполнения работы убедись, что они в порядке.
